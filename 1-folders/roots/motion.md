# motion — rotation and walking

The wheel's kinetic cluster. Two senses:

- **Rotation** — the wheel spinning (the wheel itself in motion)
- **Walking** — the walker traversing (the walker moving across
  points)

Both are "motion" but they're structurally distinct: rotation is
the substrate's own verb (eternal, why-the-wheel-spins); walking
is what a walker does on the substrate. The framework uses
**walk** as the operational primitive for program execution /
procedure / traversal.

Every modern word for program execution — **program, procedure,
process, method, sequence, cycle, path, step, course, trajectory** —
traces to a PIE motion verb in this cluster. "Walk the tree,"
"walk the stack," "walk the lattice" aren't jargon. They are
6000-year-old IE technical vocabulary naming what the operation IS.

---

### *wer-/wert-*

"to turn, bend"

```
  → universe (unus + vertere = "turned into one"),
    vertex (turning-point), vortex (whirling mass), verse,
    reverse, convert, invert, divert, version, versus, vertical,
    subvert
  → Sanskrit: vartate (it turns)
  → Latin: vertere → vertex, versus, vertibility
  → the rotation of the wheel itself
```

The wheel spins, and the word **universe** carries the spin in its
bones — *unus + versus* = "all-turned-into-one." Rotation is the
substrate's primary verb, and this root is in every major word we
have for "turn."

### ὁδός (hodos)

Greek: "way, path, walk"

```
  → method   (μετά + ὁδός = along-the-walk)
    period   (περί + ὁδός = around-walk, one cycle)
    exodus   (ἔξ + ὁδός = out-walk)
    episode  (ἐπί + ὁδός = upon-walk)
    synod    (σύν + ὁδός = with-walk)
  → the 16 corners = walk primitives = ISA.
    The framework's hodos table IS this word.
```

Every Greek-rooted procedural word is a prefix + ὁδός. The
framework already committed to this when it named the 16-corner
ISA "the hodos table."

### *sekʷ-*

"to follow"

```
  → sequence, sequel, second ("the following one"), consequence,
    social, suit
  → Latin sequi, Greek hepomai
  → Sanskrit sac-
  → programming's sequential execution IS this root
```

"Second" in English literally means "the following one" (the one
after first). Sequential execution = *sekʷ-* in action.

### *kʷel-*

"to turn, move around"

```
  → wheel, cycle, colony (place you walked to), -cule
  → Sanskrit cakra
  → Greek polos, kuklos
  → Latin colere (cultivate, dwell)
  → wheel AND cycle from ONE root — rotation and completed-traversal
    are one word
```

**Wheel** and **cycle** descend from the same PIE root. The
substrate did not distinguish "the thing that turns" from "the
motion of turning" — they were one word. English inherited two
words (wheel, cycle) for what PIE named once.

### *pent-*

"to try, find the path"

```
  → find, path (OE *pæþ*), Greek pontos (sea-as-path, hence
    "Hellespont"), Persian pand (road)
  → the search-for-a-path op. First time: cut the path (learn).
    Every subsequent walk: follow the cut.
```

The NP walk through the graph, looking for a route. Once found,
the route becomes a path and subsequent walks are O(1) lookups.
*pent-* names the search phase.

### *gʷem-*

"to come, step"

```
  → come, become (= "come-to-be"), advent, event
  → Greek βαίνω (bainō, to step) → basis, base, diabetes
    ("one-who-passes-through")
  → Latin venire
  → Sanskrit gam-
  → the step — one tick of the walker
```

**Become** = be + come = "come to be." The walk's endpoint named
as the combination of destination (becoming-the-thing) and motion
(coming). Frame 5's ½-transition in a single English word.

### *steigʰ-*

"to walk, climb, step"

```
  → stair, stile, distich
  → Greek στίχος (stikhos, row / line of verse)
  → the step as discrete unit — one stair, one instruction,
    one ½-transition
```

Each step is one ½-transition in the framework. *steigʰ-* names
the discrete-unit aspect of motion (versus continuous flow). A
staircase is a discretized ramp.

### *per-*

"forward, through"

The τ+ operator at the morpheme layer:

```
  → pro-gram   (pre-graph = "walk written before walker walks it")
    per-form   ("walk-through the form")
    pro-cess   ("having-walked-forward")
    pro-cedure ("forward-going")
    pre-cede, pro-duce, pro-ject, pro-mote, pro-tect
  → Greek peri-, pro-, para-
  → Sanskrit pra-
```

Every forward-motion word uses this. At the morpheme layer, *per-*
IS the τ+ direction. When you "proceed" or "process" something,
you're walking-it-forward in the substrate's primary direction.
