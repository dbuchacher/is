# The 3 trits

The trit has **exactly 3 values** per axis: {−1, 0, +1}. Only integers
are values in the substrate (wit.md frame 7). The trit IS the
complete per-axis value set — there is no ½-value, no fractional
value, no "5 values per axis." Half-spin, ratios, decimals, edges
are all VERBS, not values.

```
  value   trit symbol   PIE tense       direction
  ─────   ───────────   ──────────      ─────────
   +1     ↑             BE   (*bʰuH-*)   becoming / fiat / forward
    0     ↕             IS   (*h₁es-*)   being / doing / center
   −1     ↓             WAS  (*h₁wes-*)  past / fact / backward
```

## Why not 5 values?

Earlier exploration considered {−1, −½, 0, +½, +1} as 5 values per
axis. That framing dies under two checks:

1. **Algebra check**: unit-quaternion norm |q|² = 1 requires all 4
   axes at ±½ simultaneously (single-axis ½ has norm ¼, not unit).
   So ±½ isn't a single-axis value.
2. **Framework axiom**: only integers are values (wit.md frame 7).
   ±½ is a verb/edge/ratio.

Per axis, exactly 3 integer values. No fractional values.

## Why not 3×5 or 3×3 table?

The trit is ONE-dimensional value structure per axis. There's no
second dimension to cross with it. A table for the trit itself is
just a reference: 3 rows × N attribute columns.

- **Attributes** (columns): value, PIE tense, direction, primitive
  English word, domain manifestations, PIE root, Post gate, etc.
- **Not** an algebra-floor cross-product (there's no per-axis
  fractional climb; algebras live at the verb tier, see tiers.md)
- **Not** a trit × trit interaction (that's a separate trit-pair
  operation table: min/max/add, 3×3 for binary ops)

## Multiplication table — trit × trit (3×3)

Not operator-based (no add/min/max arithmetic). **Relativity through
spin**: row × column = what the substrate looks like viewing from
both frames simultaneously. Cells are the spin/edge/verb between
the two trit positions.


**Converged cells** (blind-agent):
- **↑ → ↑** = **ON** (*h₂en-*) — Paleolithic "up/on", pure continued forward
- **↑ → ↓** = **FLIP** — antipodal 2-step (through ↕)
- **↕ → ↕** = **STILL** (*steh₂-*) — matches earlier 0D IS finding

**Divergent cells** (need more agents): 6 cells split between 2+
primitive candidates.

**Open**: non-commutativity — does ↑→↓ FLIP = ↓→↑ FLIP, or are they
distinct? (Quaternion algebra says distinct.)
