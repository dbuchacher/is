# Element Mapping — Status

## What's proven

- Each physics formula IS a lattice coordinate: T^a × D^b × M^c × Q^d
  The exponent vector (b,c,d,a) = (sky,water,sun,mark). This is dimensional analysis.
- 8 SI base units + reciprocals = 8 period-1 lattice points.
- Dependency order D→M→Q→T matches Genesis and Big Bang (3 sources).
- Period structure C(4,k) = 1,4,6,4,1 = 16 cells.
- ~118 stable monomials exist. Matches element count.

## What's NOT proven

- Which chemical element maps to which formula.
- A chemical element is NOT a single monomial — hydrogen has mass AND
  size AND charge AND spectral lines. It occupies MULTIPLE lattice points.
- The prior work mapped one property per element (H=frequency, C=wave_equation).
  This is a projection, not a bijection.
- We don't have the function Z → (sky,water,sun,mark).

## What's in the codex

- Formula column: IMPORTED (verified, dimensional analysis is math)
- Element column: NOT imported (speculative, property-matching only)
- The import-elements.json file exists but has the wrong dimension ordering
  AND the element assignments are property-matches, not derivations.

## Next steps if someone picks this up

1. The element mapping is a SET mapping, not a point mapping.
   H → {D, M, Q, D²M/T², ...} = all formulas relevant to hydrogen.
2. The "which element gets which formula" question might be wrong.
   Maybe elements don't map 1:1 to formulas. Maybe elements are
   WALKS through multiple formulas, not single points.
3. The Z→coordinate function might involve the fine structure constant.
   α = 1/137 = lattice geometry. Z determines electron configuration.
   Both are governed by α. The connection might be through α, not direct.
