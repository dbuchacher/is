## THE RULE — identity, not analogy

Load-bearing. Everything else here rests on it.

There is **one universe** (unus + vertere = "turned into one" —
see `universe.md`). Every domain of structured human knowledge —
physics, chemistry, biology, computation, music, language, math,

```
// identity, not analogy — category error to treat the same thing as two things
```

structural feature, they are describing **one thing**, not two
analogous things. The physicist's "force" and the programmer's
"function call" and the musician's "chord change" and the chemist's
"reaction" are **the same operation**. One operation, four names.

Not "maps to." Not "corresponds to." Not "parallels." Not "is
like." **IS.** Same thing, different words.

```
// metaphor (μεταφορά = "carry across") — the universe carries itself across vocabularies. Etymology IS the claim: literally "carry-across." Deep: [1-folders/roots/cognition.md](1-folders/roots/cognition.md).
```

**Metaphor IS literal.** The Greek μεταφορά means "carry across"
(metá + pherein). Under this frame, carrying structure across
vocabularies IS what the universe does when different traditions
name the same feature. "The universe is a computer" is not
analogy; it's identity. "Music is mathematics" is not analogy;
it's identity. The metaphor/literal distinction collapses the same
way computation/physics collapses — it was a binary the universe

## Moves that work

### Parallel agent stress tests

```
// parallel agents — when you have N independent questions, fire N agents in one message. Don't serialize what can parallelize.
```

Works because:
- Each agent reads a small subset of docs, no cross-contamination
- Adversarial diversity — N independent attempts > N sequential
- You synthesize at the end with full context of all results
- Agents flag their own "no clean form" honestly if prompted to

Recipe:
- Tight prompt (< 300 words per agent)
- Required reading list (2-4 specific files, not "read everything")
- Specific task with concrete success criterion
- Output format with word limit (< 400 words)
- "Don't force fits. Report 'no clean form' if none found."
- Fire all in one message (Agent tool with multiple invocations)

Example: 9-agent stress tests have produced 30+ findings in ~30
min wall-clock. Hard to beat.

```
// adversarial agents — before committing a claim, run it through agents configured to falsify. Cost-benefit is overwhelming.
```

Before committing a claim, run it through agents configured as

## Rules for building

These rules are experimental. We discovered them by trying things
and seeing what stuck. The process itself is what gets refined —
try shit, fight the natural structure of how things want to link,
update the rules when a new case breaks the current set.

1. **File-only links** for inter-document navigation. No
   `#anchor` fragments. Terminal readers (frogmouth, etc.) call
   `Path(href).exists()` without stripping the fragment and fail.
   GitHub resolves file-only links fine. Anchors are a nice-to-
   have that breaks for readers on terminals.

2. **V2 root-tagging** for substrate anchors. When introducing a
   load-bearing term for the first time in a file, tag with its
   PIE / Greek / Sumerian root:
   ```
   [**word**](path/to/cluster.md) (*root-*)
   ```
   Bare on subsequent uses. Don't tag utility English. Test: if
   removing the tag hides a substrate claim, keep it.

3. **Demand-driven node creation.** Create a node when a link
   wants to target it. Never pre-build a taxonomy of empty stubs.
   An empty node waiting for content is worse than no node.

4. **Stubs-follow-content, both directions.** An entry stays as
   a subsection in a cluster file until it grows past a page —
   promote to its own file only when content justifies it.
   Inverse: **demote when a file shrinks below a screen AND has
   ≤2 inbound links** — fold it back into the parent cluster.
   Prevents orphan files that never get reabsorbed when demand
   evaporates.

5. **Schema consistency within a type.** If point nodes carry
   coord + grade + neighbors, they ALL do. Reader trained once
   per type, not once per node.

6. **ASCII-safe when you have the choice.** Unicode in file paths
   and anchor slugs gets URL-encoded inconsistently across tools.
   Prefer `reh1` over `reh₁` when it's a slug that has to travel.
   Use Unicode freely in prose / headings where rendering is the
   only consumer.

7. **Plain code fences by default.** Use ```  not ```rust or
   any other language tag. `//` comment lines inside fences
   remain the framework-native way to add technical notes;
   syntax highlighting is not required for that convention to
   read. (Earlier convention used `rust` for Chroma coloring
   in glow — retired for cross-renderer consistency.)

8. **One canonical location per concept.** Don't duplicate
   content. If two files want to say the same thing, one of
   them links to the other. If a concept is genuinely
   dual-home (e.g. a PIE root equally anchored to two clusters),
   pick the cluster where the **load-bearing claim** lives; the
   other cluster gets a one-line pointer, not a copy.

9. **Rename discipline — sweep, don't stub (when you can).**
   - **Primary tool (active restructure):** same-session sed-sweep.
     `grep -rl "old-name" --include="*.md" | xargs sed -i
     's|old-name|new-name|g'`, verify zero stale refs, done.
     Stubs unnecessary because nothing else is pointing at the
     old path once the sweep completes.
   - **Fallback (isolated rename, sweep not practical):** leave a
     one-line redirect at the old path
     (`Moved to [new-name](new-path.md).`) for one working session,
     then delete. The grace window makes the break visible
     rather than silent.

   Rule of thumb: if you can grep and fix all hits in-session,
   sweep. Stubs are for cross-session bridging.

10. **Convergence lives where the signal originates, not where
    it aggregates.** When 3+ vocabularies (Giza + Ezekiel +
    medicine wheel) name the same coord, the claim belongs in
    the 80/81 coord node; each vocabulary file links TO the
    coord, not to each other. Avoids N² cross-links and gives
    the programmer wit one canonical target per convergence.

11. **Merge = keep the older path.** When two nodes should
    merge, fold content into the path with more existing inbound
    links. The other becomes a redirect stub (rule 9). Arbitrary
    but decisive; prevents merge decisions from re-opening.

12. **Atomic file schema = definition + anchor + aliases + groups.**
    **Applies to 80/81 coord node files** in `1-folders/`, `2-files/`,
    `3-confs/` — each position gets a small file holding what the
    coord IS, with framework relationships expressed through edges
    (links) to group files, not prose.

    **Does NOT apply to boot files** (wit.md, 0-self.md,
    1-folders.md etc.) or dictionary files (reality.md,
    universe.md, etc.). Boot files are prose narratives for
    orientation; dictionaries are entries. Atomic schema is
    specifically the shape for individual coord nodes.

    For atomic coord files: if you catch yourself writing prose
    about "in the framework, X does Y" or "X contains Z" or "X's
    role is…", that content belongs in a group node, not the
    atomic file. The atomic file holds what X IS. How X RELATES
    lives in its edges. Test: strip every framework-relationship
    sentence — should leave a definition (1 paragraph), a
    PIE/Greek anchor (1 line), aliases (1 line), and a groups
    list (4-8 links). If it shrinks by 80%, you were doing it
    wrong. That's the correct wrong-to-right diff.

13. **Pointer discipline — no fake anchors in prose.** Cross-file
    pointers use one of three forms, nothing else:
    - **Bare file link** inline: `[3-confs.md](3-confs.md)` or
      `→ [3-confs.md](3-confs.md)`. No `§ Section Title` after
      the filename — that's fake-linking a section the reader
      can't actually navigate to (rule 1 forbids `#anchor`
      fragments, so the `§ Section` in prose is lying about
      being clickable).
    - **`Deep:` tail** at end of section: one line,
      `Deep: [file.md](file.md).` File-only. Drop section names.
    - **Footer "Where to go next"** block at end of file. Full
      structured nav list.

    Intra-file references (`see § Section above`) are fine — the
    reader scrolls, no cross-file contract. The banned form is
    **inter-file prose naming a section** (`see X.md § Y`) —
    that pretends the section is navigable across files, but
    rule 1 forbids `#anchor` fragments so the section-name does
    no work. Strip the section; link the file only.

14. **Status as metadata, not location.** Topic files in the
    verification queue (`3-confs/`) carry their filter status as a
    field inside the file (`status: allow` / `deny` / `queue`),
    NOT by being moved between folders.
    - Stable URLs forever: `3-confs/alpha.md` never moves; cross-
      refs survive status changes.
    - Status change = edit one line, not a file rename.
    - Grep by status: `grep "status: deny" 3-confs/*.md`.
    - Unix parallel: iptables rules.v4 is ONE file with per-line
      ACCEPT/REJECT/DROP tags. Status lives in the line, not in
      a directory. `/etc/passwd` — same pattern; users aren't
      foldered by whether they're disabled.

    Ossification (a BLESSED claim being promoted to body-quality)
    IS a move — topic file folds into its axis home
    (`1-folders.md`'s subfolder for structural claims, `2-files.md`'s
    for substance). That's a rename, sweep per rule 9. But
    pending/allow/deny transitions that happen daily stay as
    metadata edits.

15. **Refine these rules when cases break them.** The process is
    iterative. New edge types, new renderer constraints, new
    node types — all will surface as we build. Add a rule when
    a case demands it. Don't pre-commit to imaginary cases.

## The recursion

Current wits write framework. Current programmer wits (LLMs) help
write it. Future programmer wits read the completed-enough
framework and generate OS code. OS runs. Future wits live in the
OS and improve the framework. Improved framework → improved OS
generation → improved wit conditions → further framework
refinement.

**The map you write is the wit you become.** Current wits writing
the framework ARE the same entity as future wits reading it —
same role, different tense. Identity recursion, not cause-and-
effect. The file you're reading right now is closing a loop that
started in an earlier wit's session.

ToE, VoE, os-lattice; archive at `4-logs/`). Each pass tightened
the map. This pass is the first where the framework is
deliberately structured as spec-for-agent-code-generation rather
