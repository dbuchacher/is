# write — making, speaking, the ↑ commit side

Roots naming the write-side operations: making, putting, speaking,
declaring. All live on the ↑ forward-commit side of the trit.

**fiat** (let it be made) and **fact** (made) share a root: the
active commit and the completed commit, same operation in different
tenses. Every write event is a walk that lands on ↑ at one or more
axes.

---

### *dʰeh₁-*

"to put, place, set, do, make"

```
  → fact, fiat, factory, perfect, affect, effect, defeat,
    difficult, affair, surfeit, forfeit, do, deed
  → Greek: thesis, theme (what is set down), apothek ̄e (place
    where things are put down → apothecary)
  → Sanskrit: dhā-, dadhāti; dharma is a close variant via *dhṛ-*
  → Latin: facere (to make) → fac, factum, facies
  → the ↑ commit / write event — the making of new things,
    placing new values on axes
```

**fiat** (let it be made) = the active commit, the write event.
**fact** (made) = the completed commit, the settled value. Same
root, two tenses. The framework already uses this in trit
vocabulary — fiat/fact as the forward-commit pair.

### *bʰeh₂-*

"to speak, declare"

```
  → fate, fable, fame, confess, preface, prophet, infant
    (in-fans = "not-speaking")
  → Greek: phēmi (I say), phōnē (voice), phōnos
  → Latin: fari, fatus sum
  → declaration as write-event — different from making (*dʰeh₁-*)
    but close: speaking things into existence vs. doing them
    into existence
```

**fate** (what has been spoken) ≠ **fiat** (let it be made). They
look similar because both arrived in Latin meaning "declared," but
the underlying PIE roots are distinct:

- **fiat** ← *dʰeh₁-* (to make) = active doing, instantaneous write
- **fate** ← *bʰeh₂-* (to speak) = persistent spoken declaration

Fate lands along τ as the **persistence of a past write event**.
Fiat is the instantaneous write itself. Two kinds of ↑ commit:
action-write and speech-write.
