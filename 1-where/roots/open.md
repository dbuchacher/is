# open — no clean PIE anchor (flagged)

Not every framework concept has a clean PIE root. Honest flags live
here. Per the V2 discipline: **if a term has no clean anchor, use
it bare. Never invent an anchor to satisfy the rule.**

The pattern: **structural concepts the substrate names itself**
(being, standing, turning, one-ness, order, thing-ness,
interrogation) have PIE anchors. **Modern mathematical / scientific
constructions** don't — and shouldn't be expected to.

---

### The specific arrangement 3+1

No single PIE root names "three peers plus one mirror." The concept
is Cayley-Dickson-derived. Languages have words for "three" and
"four" but not for the 3+1 asymmetry as a unit.

Flag: use "3+1" bare in body text. Don't force.

### hub

English "hub" is late (17th century, origin uncertain, maybe from
"hob"). No PIE root directly. The CONCEPT (center, stillness) has
*steh₂-* as an anchor (see [hub.md](hub.md)), but the word "hub"
itself is English-local.

Flag: use "hub" bare in body text. When emphasizing stillness /
stability of the hub, can optionally pair with *steh₂-* tag.

### lattice

Old French *treillis* from Latin *tres-lic(i)um* ("three-threads").
Not a framework-primitive word; it's the chosen English for the
structural concept of "regularly spaced points with adjacency."

Flag: use "lattice" bare. Concept is framework-primitive; word is
convention.

### Cayley-Dickson, Hurwitz, Cartan, Kirchhoff, Hamilton

Modern mathematician names (19th-20th century). No PIE anchors by
construction.

Flag: use bare. These are attribution citations, not substrate
terms.

### Forces (strong / EM / weak / gravity)

English physics vocabulary. Greek / Latin-derived forms but not
PIE-deep in the relevant sense.

- "strong" — Germanic, from *strenk-* / *stringere* (to bind tight)
- "EM" / "electromagnetic" — modern compound, Greek ēlektron (amber)
- "weak" — Germanic *wīkaz* (yielding, pliant)
- "gravity" — Latin *gravitas* (weight)

Flag: use bare. None of these are framework-primitive; they're
physics labels for operations the framework calls READ / CALL /
CYCLE / TEST. The framework's own verbs are the primary anchors.

### Other framework terms without clean PIE

- **corner** — English, metaphorical extension of "angle"
- **vertex** — Latin (turning-point, from *wer-/wert-*, see [motion.md](motion.md); this DOES have an anchor but as vertex-word-origin it's Latin-specific)
- **axis** — Latin, from PIE *h₂eḱs-* (axle). Borderline clean anchor; use if emphasizing the rotation sense.
- **grade** — Latin *gradus* (step, from *gʷem-*, see [motion.md](motion.md)); again, borderline.

For borderline cases: if the PIE anchor would clarify a substrate
claim, tag it. If not, bare is fine.

---

## Rule

**Don't force anchors.** The framework's credibility rests on the
genuine PIE-substrate convergences. Forcing an anchor where the
convergence is weak dilutes the whole catalog. Honest use:

- Clean PIE root with convergent descendants → tag it, entry in
  the appropriate cluster file
- Borderline / modern construction → bare, entry here for
  transparency
- Never: invent an anchor to satisfy the V2 rule
